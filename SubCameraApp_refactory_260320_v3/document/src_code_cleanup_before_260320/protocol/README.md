# Protocol Definitions Layer

## 📁 폴더 개요
본 폴더는 시스템 전반에서 통용되는 공통 데이터 규약과 패킷 구조를 정의합니다. 네트워크 레이어와 비즈니스 레이어 사이의 데이터 해석 기준을 제공합니다.

## 🛠 문제 해결 및 최적화
고성능 바이너리 통신을 위해 다음과 같은 설계가 적용되었습니다.

1.  **경량 패킷 헤더 (PacketProtocol)**:
    - 부하를 최소화하기 위해 필수 정보(Magic, Type, Length)만을 포함하는 바이너리 헤더를 정의하여 통신 오버헤드를 줄였습니다.
2.  **메시지 타입 추상화**:
    - 이미지 전송, AI 통계, 장치 제어 등 다양한 메시지 유형을 Enum으로 정의하여 코드 전반의 가독성과 유지보수성을 확보했습니다.

## 🔄 데이터 흐름 (Data Flow)
- **Shared Reference**: 본 폴더의 헤더 파일은 `network`, `transmitter`, `controller` 모듈에서 공통으로 참조하여 패킷 생성 및 해석에 사용됩니다.

---

## 🔍 코드 품질 분석 (Code Review)

### ✅ SOLID 원칙 점검
- **SRP (Single Responsibility)**: 데이터 구조 정의에만 충실하여 다른 모듈의 로직에 간섭하지 않습니다.

### ⚠️ 코드 클린업 대상
- **Endianness Support**: 현재 구조체 직접 매핑 방식은 서로 다른 엔디안(Endian)을 가진 장치 간 통신 시 문제가 될 수 있습니다. 네트워크 바이트 오더링(htons/ntohs 등)을 적용하거나 프로토콜 버퍼(Protobuf)와 같은 직렬화 도구 도입을 검토할 수 있습니다.
