# Data Buffer Layer (Event Recording)

## 📁 폴더 개요
본 폴더는 실시간으로 유입되는 영상 데이터를 임시 관리하고, 특정 이벤트(낙상) 발생 시 과거와 미래의 시점을 결합하여 클립을 생성하는 **데이터 지속성(Persistence) 관리자**입니다.

## 🛠 문제 해결 및 최적화
이벤트 발생 전의 "과거 상황"을 복고풍으로 추적하기 위해 다음과 같은 최적화가 적용되었습니다.

1.  **순환 버퍼 (Circular Buffer)**:
    - 메모리 효율성을 위해 고정된 크기의 배열을 순환하며 최신 N초 분량의 프레임을 유지합니다. 별도의 메모리 재할당 없이 안정적인 버퍼링을 제공합니다.
2.  **비동기 스냅샷 처리**:
    - 이벤트 발생 시 메인 처리 스레드를 멈추지 않고, 버퍼의 내용을 즉시 복사하여 별도의 `transmit_thread`에서 처리를 위임함으로써 실시간 파이프라인의 레이턴시(Jitter)를 최소화했습니다.
3.  **샘플링 FPS 조정**:
    - 저장 용량 최적화를 위해 실시간 스트리밍 FPS보다 낮은 주기(예: 1 FPS)로 버퍼링을 수행하도록 `AppConfig` 연동 설계가 되어 있습니다.

## 🔄 데이터 흐름 (Data Flow)
- **Input**: `StreamPipeline`으로부터 매 초 전달되는 영상 프레임.
- **Output**: 
  - 로컬 스토리지: `.jpg` 또는 영상 파일 저장.
  - `transmitter`: 생성된 클립 데이터를 서버로 전송 대기열로 삽입.

## 🔍 코드 품질 분석 (Code Review)

### ✅ SOLID 원칙 점검
- **SRP (Single Responsibility)**: `CircularFrameBuffer`는 오직 보관에만, `EventRecorder`는 오직 이벤트 로직에만 충실합니다.

### ⚠️ 코드 클린업 대상
- **Thread Lifetime Management**: `EventRecorder` 내부에서 별도 스레드를 생성하여 전송을 처리할 때, 소멸자에서 해당 스레드의 안전한 종료(`join`) 여부를 확인하는 로직을 더 견고하게 다듬을 필요가 있습니다.
