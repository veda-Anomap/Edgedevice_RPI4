# Rendering & Visualization Layer

## 📁 폴더 개요
본 폴더는 최종 사용자에게 보여질 시각적 요소를 처리합니다. AI 분석 결과(박스, 관절, 텍스트)를 영상 위에 겹쳐 그려서(Overlay) 정보를 직관적으로 전달하는 역할을 수행합니다.

## 🛠 문제 해결 및 최적화
시각적 정보를 효율적으로 표현하기 위해 다음과 같은 기법이 적용되었습니다.

1.  **전용 렌더링 인터페이스**:
    - 영상 처리 스레드와 렌더링 로직을 분리하여 `FrameRenderer`가 오직 그리기 작업에만 집중할 수 있도록 구성했습니다.
2.  **Batch Drawing**:
    - 다수의 객체가 감지되었을 때, 리스트를 순회하며 효과적으로 드로잉 연산을 수행하여 연산 낭비를 줄였습니다.
3.  **상태 기반 텍스트 강조**:
    - "Falling"과 같은 위급 상황 발생 시 폰트 색상을 변경하거나 굵기를 조절하여 가독성을 높이는 사용자 중심 디자인이 적용되었습니다.

## 🔄 데이터 흐름 (Data Flow)
- **Input**: `StreamPipeline`으로부터 전달된 영상 프레임 및 AI 감지 결과(`DetectionResult`).
- **Output**: 
  - 시각적 오버레이가 완료된 최종 프레임 (네트워크 송출 또는 로컬 디스플레이용).

---

## 🔍 코드 품질 분석 (Code Review)

### ✅ SOLID 원칙 점검
- **SRP (Single Responsibility)**: 렌더러는 분석을 하지 않고 오직 '그리는 행위'에만 독립되어 있어 원칙을 매우 우수하게 준수하고 있습니다.

### ⚠️ 코드 클린업 대상
- **Dynamic Configuration**: 현재 텍스트 크기나 선의 굵기가 하드코딩된 부분이 있습니다. 이를 사용자 설정이나 해상도 비율에 맞게 동적으로 조정하는 로직을 추가하면 다양한 디스플레이 환경에 대응하기 좋습니다.
