# 낙상 감지 알고리즘 설계 비교 보고서 (Design Comparison)

본 문서는 `SubCameraApp`에 적용된 기존(Legacy) 낙상 감지 방식과 새롭게 도입된 **물리 기반 상태 머신(Physics-based State Machine)** 방식의 기술적 차이와 장단점을 분석합니다.

---

## 1. 알고리즘 설계 요약

### [Algorithm A] 레거시 (Rule-based Voting)
- **핵심 지표**: 픽셀 단위 중심점 $y$값 변화량 ($\Delta y = y_{curr} - y_{prev}$).
- **판정 방식**: 속도, 각도, 압축률 조건이 맞으면 매 프레임 '투표(Vote)'를 던지고, 윈도우 내 투표 수가 일정 이상이면 낙상으로 확정.
- **특징**: 단순 비정상 자세 감지 (Anomaly Detection) 위주.

### [Algorithm B] 고도화 (Physics + State Machine)
- **핵심 지표**: 몸통 길이 기준 정규화 속도 ($V_{norm} = \frac{\Delta y / \Delta t}{TorsoLength}$).
- **판정 방식**: [STANDING → FALLING → LYING_DOWN]으로 이어지는 시퀀스 추적 및 최종 '정지(Stillness)' 검증.
- **특징**: 물리적 인과관계 및 시간적 흐름(Temporal Logic) 기반.

---

## 2. 상세 비교 테이블

| 비교 항목 | [A] 레거시 방식 | [B] 고도화 방식 | 비고 |
| :--- | :--- | :--- | :--- |
| **속도 계산** | $\Delta y$ (Pixel/Frame) | $\Delta y / \Delta t / Torso$ | 물리적 절대 속도 산출 |
| **FPS 변화 대응** | 취약 (FPS 하락 시 속도 급증 오판) | **강건** (시간차 $\Delta t$로 보정) | RPi 4B 필수 기능 |
| **원근(Scale) 대응** | 취약 (가까우면 속도 과대평가) | **강건** (몸통 길이로 정규화) | 거리 무관 일관성 유지 |
| **오탐 방지 (앉기)** | 각도/주요 관절 순서 위주 (불안정) | **사후 정지 상태 검증** (안정적) | 앉은 후 움직임 유무 판정 |
| **연산 부하** | 매우 낮음 | 낮음 (정규화/필터링 추가) | 성능 차이 미미 |
| **지연 시간** | 낮음 (즉시 판정 가능) | 약간 높음 (1~1.5초 정지 확인) | 신뢰도를 위해 지연 감수 |

---

## 3. 시나리오별 장단점 분석

### 시나리오 1: 빠른 이동 및 뛰기 (Fast Walking/Running)
- **[A] 방식**: 수직 이동량이 크면 낙상으로 오판할 확률이 매우 높음.
- **[B] 방식**: 속도가 빠르더라도 바닥에 도달한 후 '머무르는(Stillness)' 과정이 없으므로 낙상으로 판정하지 않음 (**장점**).

### 시나리오 2: 의자나 침대에 앉기/눕기
- **[A] 방식**: 상체가 숙여지며 $y$값이 변하는 순간 낙상으로 오판 가능성 있음.
- **[B] 방식**: 머리가 위치하는 최종 $y$ 좌표가 'Ground Zone'(바닥 영역)이 아니거나, 앉은 뒤 안정적인 자세가 확인되면 차단함 (**장점**).

### 시나리오 3: 카메라 노이즈 및 스켈레톤 지터링 (Jittering)
- **[A] 방식**: 한 프레임만 튀어도 투표 시스템에 영향을 주어 불안정.
- **[B] 방식**: 몸통 길이를 EMA(지수 이동 평균)로 필터링하고 상태 전이(State Transition)를 확인하므로 소수 프레임의 노이즈를 효과적으로 무시함 (**장점**).

---

## 4. 최종 판별 및 추천

### 권장 환경
- **Algorithm A**: 연산력이 극도로 제한되어 단순 좌표 비교만 가능한 MCU 환경.
- **Algorithm B**: **라즈베리파이 4B, 윈도우 등 Edge AI 환경.** 
  - FPS가 불안정하고 카메라와의 거리가 유동적인 실사용 환경에서 훨씬 높은 신뢰도를 보장합니다.

### 결론
새로운 알고리즘은 **"낙상은 단순히 빠른 움직임이 아니라, 물리적 낙하 후 바닥에서 정지된 상태가 지속되는 일련의 사건"**으로 정의합니다. 이는 오탐(False Positive)을 획기적으로 줄여 보안 관제 시스템의 실효성을 극대화합니다.
